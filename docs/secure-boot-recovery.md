# Restoring Secure Boot after a BIOS Update

When updating your motherboard's BIOS, the Secure Boot variables (PK, KEK, db, dbx) are usually wiped. This puts the motherboard back into "Setup Mode" and disables Secure Boot. 

Since your Secure Boot keys are already generated by `sbctl` (you have an Owner GUID) and you are using **Lanzaboote** in your NixOS configuration, follow these steps to restore Secure Boot functionality.

## 1. Check Current Status
First, confirm that your keys exist but are not enrolled, and that Secure Boot is disabled:
```bash
sudo sbctl status
```
You should see:
- `Owner GUID`: (Your GUID here)
- `Setup Mode`: ✗ Enabled
- `Secure Boot`: ✗ Disabled

## 2. Re-enroll Keys into Firmware
Inject your `sbctl` keys back into the motherboard's EFI variables. We include the `--microsoft` flag to ensure vendor firmware (like your GPU) continues to load properly:
```bash
sudo sbctl enroll-keys --microsoft
```
*Note: If you get an error about "immutable files", run `sudo chattr -i /sys/firmware/efi/efivars/*` first.*

## 3. Rebuild NixOS (Lanzaboote Signing)
Lanzaboote hooks into the NixOS rebuild process to automatically sign the unified EFI kernels and the `systemd-boot` bootloader. Run a rebuild to ensure everything is freshly signed:
```bash
sudo nixos-rebuild switch --flake ".#ryzen-9950x3d"
```

## 4. Verify Boot Files
Check the signing status of the files in your `/boot` partition:
```bash
sudo sbctl verify
```
You want to see green checkmarks (`✓`) next to:
- `/boot/EFI/BOOT/BOOTX64.EFI`
- `/boot/EFI/systemd/systemd-bootx64.efi`
- `/boot/EFI/Linux/nixos-generation-*.efi`

*(Optional: If you see a single red `✗` next to an old, non-unified `/boot/EFI/nixos/kernel-*.efi`, you can safely ignore it, as Lanzaboote boots the unified images. To manually sign it for a fully green output, run: `sudo sbctl sign -s /boot/EFI/nixos/kernel-*.efi`)*

## 5. Enable Secure Boot in BIOS
Now that the firmware has your keys and your bootloader is signed, you must turn Secure Boot back on in the BIOS:

1. Reboot your PC and enter the **BIOS/UEFI**.
2. Navigate to the **Boot / Security** section.
3. Locate **Secure Boot** and change it from disabled/setup mode to **Enabled** (or **Windows UEFI mode** on ASUS motherboards).
4. Save and Exit.

## 6. Final Verification
Boot into NixOS and run the final check. It should now show a completely locked down system:
```bash
sudo sbctl status
```
- `Setup Mode`: ✓ Disabled
- `Secure Boot`: ✓ Enabled
- `Vendor Keys`: microsoft
